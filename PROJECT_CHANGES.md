## 5. Architecture & data flow

How the pieces talk to each other. Update when a flow changes.

### Content (MDX)
`content/{projects,blog}/*.mdx` → Velite build step → typed objects in `.velite/` → imported by route segments. Frontmatter validated by Zod schema in `lib/mdx/schema.ts`.

### Contact form
Browser → `react-hook-form` validates against `lib/schemas/contact.ts` → POST `/api/contact` → server re-parses same schema → honeypot check → rate-limit → Resend → return generic success/error.

### Rendering boundary
RSC by default. `"use client"` only at leaves needing state/effects/events (form, motion wrapper, theme toggle). Never at layout level.

### Home page hero (`PixelEarthStage`)
Lives in `components/pixel-earth-stage/`. Wired into `app/(site)/page.tsx`.
- `pixel-earth-stage.tsx` — client wrapper. Section is `h-[100svh]` (explicit, not `min-h-*`, so the R3F canvas's `height: 100%` resolves cleanly). Has TWO interaction zones:
  - **Globe zone** (the section element itself): `onPointerDown/Move/Up` for drag-to-spin only. No section nav.
  - **Screen zone** (an invisible `<div>` at `inset-x-0 top-0 h-[36%]`): `onPointerDown/Move/Up + onWheel` for screen-carousel rotation. Calls `stopPropagation` so the globe handler doesn't double-fire when the user grabs the screen. `setPointerCapture` so a drag that starts on the screen stays a screen gesture even if the finger crosses into the globe area.
  - Keyboard (`onKeyDown` on the section): ArrowLeft/Right rotate the screen by one section step, works regardless of which zone the cursor is over.
  - `tabIndex={0}` for keyboard focus. `aria-label` includes the active section title so screen readers announce it. Loads the heavy R3F canvas via `next/dynamic({ ssr: false })`. Renders a CSS nebula layer + vignette behind the canvas, the screen-zone overlay above it, and a film-grain overlay above everything. **No HTML title overlay** — titles are painted into a canvas texture and rendered as 3D geometry inside the canvas (see `labels-ring.tsx`).
- `lib/sections.ts` — typed section config (`SECTIONS` array of 4: About, Projects, Research, Contact). Each section has an `angle` (radians around the screen-cylinder Y axis — About 0, Projects π/2, Research π, Contact 3π/2). `SectionEnv` has optional slots (`ringColor`, `beamPalette`, `ambientHue`, `starsIntensity`) reserved for a future env driver — empty `{}` for now.
- `lib/hooks/use-section-navigator.ts` — owns all gesture logic AND drives the 3D screen rotation. State: `activeIdx` (React) + `activeIdxRef` (sync mirror so `advance()` keeps empty deps). Refs: `targetRotationRef` (radians, the rotation target for the screen group), `cooldownUntilRef`, `swipeStartRef` (records `clientX` + rotation at drag start), `wheelAccumRef`. Three gesture paths:
  - **Drag**: `onPointerSwipeStart` records the rotation at touchdown; `onPointerSwipeMove` continuously sets `targetRotationRef.current = startRotation + dx × 0.006` (same sensitivity as the globe drag); `onPointerSwipeEnd` snaps the target to the nearest multiple of π/2 and updates `activeIdx`. No mid-drag cooldown — the user can drag continuously.
  - **Wheel**: passive React handler, accumulator decays after 100ms idle, threshold 120; on threshold cross, calls `advance(dir)` which shifts `targetRotationRef` by ∓π/2. Short-circuits during the 520ms cooldown so the page can scroll past the hero freely.
  - **Keyboard**: ArrowLeft/Right call `advance(±1)`.
  - Active index is derived from rotation via `idx = round(-rotation / (π/2)) mod 4` — only updated on snap/advance, so it matches the snapped position, not the mid-drag rotation.
- `stage-canvas.tsx` — R3F `<Canvas>` host. `alpha: true` so the CSS nebula shows through. Camera `[0, 1.75, 5.6]` fov `62` — wide enough to fit globe `r=1.52` horizontally on ~9:19 portrait while keeping the screen, ray cones, and cable tops in frame. Threads `screenRotationTargetRef` (from the navigator hook) down to `<Scene>`.
- `scene.tsx` — composes the scene: lights, fog (`3.0 → 7.2`), stars (900 count), shooting stars, camera rig, dome + atmosphere + glow ring + screen-assembly, stage beams, character. Stage group sits at `STAGE_Y = 0`. No solid `<color attach="background">` — canvas is transparent. **`<ArenaScreen>` is no longer rendered directly here** — it's wrapped inside `<ScreenAssembly>` along with `<LabelsRing>`.
- `screen-assembly.tsx` — rotating `<group>` wrapping `<ArenaScreen>` + `<LabelsRing>`. `useFrame` lerps `group.rotation.y` toward `targetRotationRef.current` at rate 8/sec (smooth, matches the globe's drag-and-ease character). Reduced-motion snaps instantly. Spotlight rays in `<StageBeams>` are NOT inside this group — they keep their independent 0.06 rad/s orbit, so the spotlight visual stays exactly as before.
- `labels-ring.tsx` — open `cylinderGeometry` at `RADIUS=1.785` (just outside the arena screen's 1.78 to avoid z-fighting), same `HEIGHT=0.85` and `Y=2.85`. `MeshBasicMaterial` with the labels canvas texture, `side: FrontSide`, `transparent: true`, `depthWrite: false`. FrontSide means only the camera-facing arc of labels renders — back labels naturally occlude themselves, giving the carousel its read. The labels texture is re-painted once `document.fonts.ready` resolves so the first frame isn't stuck on the Georgia fallback.
- `camera-rig.tsx` — gentle mouse parallax. Tracks pointer, eases camera ±0.12 on x / ±0.08 on y around base y `1.75`. Look-at `(0, 1.45, 0)` for a subtle downward tilt onto the globe. Bypassed when reduced motion.
- `dome.tsx` — upper-hemisphere earth, radius `1.52`, segments `64×32`. Eases toward target rotation, auto-rotates after `RESUME_DELAY_MS`.
- `dome-rim-glow.tsx` + `dome-smoke.tsx` — replace the old `atmosphere.tsx` fresnel shader. `dome-rim-glow` paints a soft additive halo around the dome silhouette; `dome-smoke` layers a few additive billboard sprites near the base for stage fog. Both read `isGlobeEnvironment` and hide cleanly when off-section.
- `dome-lift-group.tsx` — wrapper `<group>` placed inside `scene.tsx` around the existing `<Dome />` + glow components. Lerps Y +0.8 + adds a slow Y rotation when About is entered, so the dome rises overhead as the reveal plays. The dome itself stays untouched.
- `monolith-mountain-stage.tsx` + `mountain-facet-details.tsx` + `peak-character.tsx` — Projects environment. Carved-stone mountain with three tiered faces, geometric facets, and a small peak character. Reads `isPyramidEnvironment` for visibility.
- `research-bridge-stage.tsx` — Research environment. Long suspended glass bridge with painterly cloud backdrop and silhouette figures inside. Reads `isResearchEnvironment`.
- `contact-house-stage.tsx` — Contact environment. Weathered church-house on a dirt mound with a porch, multi-pane windows, and a rooftop cross. Reads `isContactEnvironment`.
- `intro-loader.tsx` — full-screen DOM overlay that masks the canvas mount + shader compilation. Spinning SVG rings, pulsing dot, corner text, ~1.5–6s fade. ESC skips.
- `glow-ring.tsx` — equator ring radii: filament `1.53`, mid halo `1.58`, outer bloom `1.71`, flat scatter `1.79→2.46`. Breathes + shimmers as before.
- `pixel-character.tsx` — lighthouse sprite at `y=1.57`, scale `0.42`.
- `arena-screen.tsx` — floating short cylinder at `y=2.85`, radius `1.78`, height `0.85`. Inner face shows a procedural scrolling texture (vertical gradient + 6 pooled warm/cool radial lights + faint scanlines), darker scrim outside, pulsing additive rims, halo disc beneath for light spill onto the dome, and six suspension cables (length `1.0`, anchored at the top rim, fully visible) with rim-anchor stubs.
- `stage-beams.tsx` — Five spotlight pools, each rendered as a stacked **halo + hot core** pair of additive `CircleGeometry` discs tangent to the `r=1.52` sphere at raycast intersection points, PLUS a faint additive truncated-cone "projection ray" connecting the screen underside (`y=2.425`) down to the lit point so the screen reads as the source of each light. Ray opacity `0.04–0.07` (extremely subtle), narrow top (near screen) widening toward the lit pool. Core radius `0.47–0.89`, halo `0.73–1.26`. All meshes additive, depth-tested, `side: DoubleSide`, ordered via `renderOrder`. Parent group rotates `0.06 rad/s` so pools + rays orbit together; dome spins independently. Lit points: `(±0.859, 1.241, 0.190)`, `(±0.379, 1.230, -0.804)`, `(0, 1.52, 0)`.
- `glow-ring.tsx` — four concentric additive-blended discs/tori (filament, mid, outer, flat scatter). Breathes (scale + opacity), inner filament shimmers slightly. Reads `reducedMotion`.
- `stage-beams.tsx` — five cone beams driven by a config table. Each beam pulses opacity at its own speed/phase. Group rotates slowly.
- `shooting-stars.tsx` — two pre-built `THREE.Line` streaks. Each waits a random interval, picks a spawn far behind the planet, animates a short trail in/out, then resets. Disposed on unmount.
- `pixel-character.tsx` — billboarded `<Sprite>` at the north pole. 4-frame spritesheet, swaps frame every `FRAME_MS`. Bobs vertically when motion isn't reduced.
- `textures.ts` — procedural earth (128×64, equirectangular) and character (128×32) textures, both with `NearestFilter`. Continents laid out at correct lat/lon (northern + southern hemisphere). Only the **top half** of the canvas (northern hemisphere content) is sampled by the upper-hemisphere dome via `tex.offset.y=0.5, tex.repeat.y=0.5` — without this remap the bottom of the canvas would stretch over the equator silhouette of the dome and produce a white "Antarctica band." Tiny north-polar ice cap (rows 0–2) remains; bottom Antarctica band removed. Also adds `createSpotlightGlow()` — soft radial-gradient CanvasTexture used as the patch map. **And `createLabelsTexture()`**: 4096×320 canvas, paints each `SECTIONS` title at its canvas U-position (`section.angle / 2π`) using `ctx.font = "400 132px var(--font-dm-serif)"` with `letterSpacing: 30px`, `textAlign: center`, white fill (`#f3f6ff`), 22px drop-shadow. Draws each title three times (at `x`, `x ± canvas.width`) so titles straddling the canvas seam render fully. Returns `{ texture, redraw }`; `redraw()` is called by `<LabelsRing>` once `document.fonts.ready` resolves.
- `app/globals.css` — `.stage-nebula` (layered radial gradients, blue + pink + violet pooled lights), `.stage-vignette` (corner darken via multiply blend), `.stage-grain` (inline SVG fractal-noise, overlay blend, ~7% opacity, hidden in reduced motion).
- `lib/hooks/use-reduced-motion.ts` — `useSyncExternalStore` over `matchMedia('(prefers-reduced-motion: reduce)')`. Every animated piece reads it and bails when reduced.

### Section content reveal
What happens when a user presses ENTER on a centered section. Independent of the carousel — the carousel rotation freezes while a section is entered.

- **State.** `useEnteredSection()` (`lib/hooks/use-entered-section.ts`) owns `enteredSectionId: SectionId | null` + `enter()` / `exit()`. Lifted to `pixel-earth-stage.tsx`.
- **Trigger.** `<EnterChip />` (`components/pixel-earth-stage/enter-chip.tsx`) — small HTML overlay below the centered title. Renders `ENTER ⏎` when the carousel has settled on a section with a wired reveal, `OFF-AIR` when the section is unbuilt. Click/tap or `Enter` key activates. `Escape` exits.
- **Stage cue.** `stage-beams.tsx` accepts a `cueMode: "default" | "entered"` prop. When `"entered"`, each `SoftBeamCone` / `SoftBeamCoreCone` lerps its `uIntensity` uniform by a per-frame multiplier — side beams ×0.30, center key light ×1.10. Lerp rate ≈ 6/sec gives a ~0.25s cue ramp. Reduced-motion snaps instantly.
- **Screen title flash.** `arena-screen.tsx` accepts `entered: boolean`, `enteredHeader: string`, `enteredSubhead: string`. While entered, the texture scroll freezes, the screen's brightness dips through a half-second swap, and the rendered map switches to `createTitleFlashTexture(header, sub)` — a two-line set-list card layout (`PERFORMANCE 0X` over `// SECTION`). The flash texture is cached per `header+sub` pair. `screen-assembly.tsx` skips its rotation lerp while entered so the screen holds still.
- **Camera entry pose.** `lib/section-camera-targets.ts` maps each `SectionId` → `{ position, lookAt }`. `camera-rig.tsx` accepts `enteredCameraTarget: CameraTarget | null` and lerps position + lookAt toward the target (rate ≈ 2.4/sec) when set, parallax base otherwise.
- **Per-section palette.** `lib/section-palette.ts` maps each `SectionId` → `{ rim, emissive }` accent colors that the props pick up — About amber, Projects cool blue, Research deep red, Contact pale gold. The locked environments don't read this; only the reveal props do.
- **Content data.** `lib/section-content.ts` — typed `SECTION_CONTENT: Record<SectionId, { slabs: Slab[] }>`. Each `Slab` is `{ heading, body, meta }`. About + Projects have 3 placeholder entries each; Research + Contact are empty arrays for now.
- **Prop ring.** `section-reveals/section-reveal-host.tsx` dispatches by `enteredSectionId`. Each section has its own `*-reveal.tsx` that orchestrates three props in a fan around the character (angles `[-π/3, 0, π/3]`, radius `1.7`, chest-height `y=1.85`). Entry animates props rising from `y=-0.6` with cubic-out + small overshoot, staggered 0.12s per prop. Each reveal is wrapped in a delayed-unmount gate (`shouldMount` state with a 1.6s tail after exit) so the heavy prop tree doesn't mount — and the troika SDF font load can't suspend the canvas — until the section is actually entered.
- **Content ring rotation.** `lib/hooks/use-content-ring-rotation.ts` mirrors `use-section-navigator.ts` for the prop ring (3 stops instead of 4). While entered, the screen-zone pointer/wheel handlers in `pixel-earth-stage.tsx` redirect from the carousel to the content ring — same gesture vocabulary, new target.
- **Props (per section).**
  - **About → cassettes.** `about-globe-reveal.tsx` mounts three `AboutCassette` boxes (real-cassette 1.56:1 aspect, off-white shell). Each cassette's front face is a single baked canvas texture from `createCassetteFaceTexture(seed, trackNumber, heading, body, meta, ownerName)`: top owner name in tracked serif caps with a small red heart, the slab keyword as subtitle, vertical `SIDE A/B/C` marker, two gear-toothed reel cutouts with brown wound tape + spoked hubs + center pin, central tape window with `100  50  0` counter ticks, tiny legal-print body copy, multicolor printer-test stripe, three Phillips screws (two top corners + one bottom center), and a row of bottom access slots.
  - **Projects → CD-in-cases.** `projects-mountain-reveal.tsx` mounts three `ProjectsCdCase` boxes (square, translucent shell). Each face is `createCdCaseFaceTexture(seed, trackNumber, heading, body, meta)`: translucent case with mold-clip details, top `▼ INSERT THIS END`, `80` capacity stamp, centered iridescent disc (conic-gradient rainbow + 110 concentric data-track rings + outer edge highlight + silver/chrome hub with machined grooves + center hole), vertical `RECORDABLE MD` on the right side, per-disc randomized date/serial stamp, MD-style logo block at the bottom, and a lavender sticker band wrapping across the middle-right of the case carrying `TRACK 0X`, the project heading in heavy condensed caps, the body paragraph, and the meta line in tracked caps.
  - **Research → vinyl LPs (planned).**
  - **Contact → backstage laminates (planned).**
- **Exit.** Reverses everything — beam multipliers return to 1.0, screen brightness lifts and the base scrolling texture comes back, camera lerps back to the parallax pose, prop ring descends, ENTER chip reappears. `END SET ESC` button (`pixel-earth-stage.tsx`) is the explicit exit affordance.

### Locked elements
Per the working agreement when the reveal system was designed: the four environment files (`dome.tsx`, `monolith-mountain-stage.tsx`, `research-bridge-stage.tsx`, `contact-house-stage.tsx`), `pixel-character.tsx`, `peak-character.tsx`, `labels-ring.tsx`, the arena screen's visual material, and the carousel logic in `use-section-navigator.ts` are not touched by the reveal system. New behavior is added via wrapper groups (`DomeLiftGroup`), additive prop refs (`cueMode`, `enteredCameraTarget`), and side-by-side mounts (`SectionRevealHost`).

## 6. Decision log

Non-obvious choices and the reason. New decisions go at the top.

- **2026-05-14 — About cassette detail mode is a physical J-card reader.** Selecting a cassette first opens the plastic case, then the camera pushes into the same cream J-card insert until it reads like the page. The content uses a Drei `<Html transform>` layer anchored to that insert, not a detached modal or separate paper overlay, so it moves with the case. The print scale is responsive because the close camera crop that feels right on desktop would cut text off on mobile.
- **2026-05-14 — Each section's content reveal is a physical music-format prop, not a generic content card.** About reveals cassettes, Projects reveals CD-in-cases, Research will reveal vinyl LPs, Contact will reveal backstage laminates. Iterated through three prior designs first (concrete monoliths, then "listening-set" stage props with track stencils, then album-cover treatments). None of those felt distinctively tour-coded enough. The music-format direction finally landed: it leans into the discography metaphor without touching anyone's IP, every section gets its own visual identity, and the props feel like real objects from a touring artist's universe instead of UI cards.
- **2026-05-14 — Cassette / CD-case heading typography is canvas-baked (Impact), not drei `<Text>`.** Drei's default troika SDF font lacks the brutalist condensed weight the brand wants. Baking the heading into the cassette / CD face's canvas texture lets us use real Impact / Haettenschweiler with letter-spacing, double-strike marker bleed, and per-letter wobble. The face texture canvas is 1024px-wide so the type stays crisp at typical entry-camera distance. Drei `<Text>` still earns its place on the early slab prototype's body copy, but on the cassette + CD it's gone entirely.
- **2026-05-14 — Reveal components are gated behind a delayed-unmount state flag.** Drei `<Text>` from `@react-three/drei` lazy-loads the troika SDF font on first mount, which suspends the surrounding `<Canvas>`'s Suspense boundary — blanking globe, screen, beams, environments, everything. Wrapping Text in its own `<Suspense fallback={null}>` prevents the blank but doesn't stop the suspension from being expensive. Real fix: don't mount the reveal subtree until the user actually enters the section. `AboutGlobeReveal` / `ProjectsMountainReveal` are wrapped in a tiny gate that sets `shouldMount=true` when `entered` flips on, and keeps it true for 1600ms after `entered` flips off so the exit animation can complete before unmount.
- **2026-05-14 — Reveal system never touches the four environment files.** Behavior is added via wrapper groups (`DomeLiftGroup` lifts the dome on About entry without touching `dome.tsx`), additive props (`stage-beams.tsx` gains `cueMode`, `camera-rig.tsx` gains `enteredCameraTarget`, `arena-screen.tsx` gains `entered/enteredHeader/enteredSubhead`), and side-by-side mounts (`<SectionRevealHost />` sits next to the locked elements in `scene.tsx`). Reason: each environment is a tuned piece of art whose lighting / cable / smoke / glow values are non-trivial to recreate. Wrapping is reversible, editing is not.

- **2026-05-13 — Title font: DM Serif Display (`--font-dm-serif`).** Iterated through Geist Mono (felt like a build log), Instrument Serif (close but editorial-leaning), then Fraunces with `opsz 144 / SOFT 100` (good but still not quite right). Settled on **DM Serif Display** — purpose-built display Didone with heavy verticals that hold up under Canvas2D rendering + cylinder curvature, high contrast that reads as cinematic-marquee (Italian classical film-title DNA), and distinctly less overused than Playfair Display or Bodoni proper. Loaded via the same `next/font/google` pattern as Geist Sans/Mono — no new dep. Canvas tuning: 132px size + 30px letter-spacing + 22px shadow blur, all calibrated for DM Serif's denser glyph proportions versus the prior Fraunces-era values.
- **2026-05-13 — Section titles are now 3D, painted onto a canvas texture wrapped around the screen cylinder.** Prior attempts: (1) drei `<Text>` inside the canvas — crashed the WebGL context under R3F 9.6.1 + React 19; (2) HTML overlay `<motion.h2>` floating in front of the canvas — worked but the user explicitly wanted the title to *rotate with* the screen, not "crossfade in the center." Resolution: `createLabelsTexture()` paints the four titles onto a 4096×320 canvas at the correct U-positions (`section.angle / 2π`), wrapped around a `<LabelsRing>` cylinder at `RADIUS = 1.785` (just outside the screen's 1.78 to dodge z-fight). Both cylinders live inside `<ScreenAssembly>` whose `rotation.y` is lerped toward `targetRotationRef.current` — so the screen + labels rotate together as one physical object. `MeshBasicMaterial` with `FrontSide` means back labels self-occlude (no see-through to mirrored text), which gives the carousel its natural "out-of-view-then-into-view" reveal.
- **2026-05-13 — Screen rotation is a continuous drag with snap-on-release, NOT a discrete advance.** The previous swipe-threshold model felt unphysical — the screen didn't react until threshold was crossed, then "popped" to the next section. New model: `onPointerSwipeMove` continuously sets `targetRotationRef.current = startRotation + dx × 0.006` (same sensitivity as the globe drag) — the screen tracks the finger 1:1 via lerp. `onPointerSwipeEnd` snaps to nearest π/2 multiple; the lerp animates the snap. Wheel + ArrowLeft/Right still call `advance(dir)` which shifts the rotation target by ∓π/2. `activeIdx` is derived from the snapped rotation, so React state only changes when the user commits — no jitter during the drag itself.
- **2026-05-13 — Spotlight rays kept independent of the screen rotation group.** Could've nested `<StageBeams>` inside `<ScreenAssembly>` so rays follow the screen. Didn't, because the user explicitly said "keep the spotlight intact" and the existing 0.06 rad/s independent orbit IS the spotlight look. The rays come from `y=2.425` (the screen underside in WORLD space) regardless of screen rotation, which still reads as "screen-sourced light" because the rays' top points stay aligned with the static screen position even when the screen geometry above rotates around the Y axis.
- **2026-05-12 — Globe rotation and section nav are bound to SEPARATE pointer/wheel zones, not the same root element.** Initial attempt routed every gesture through the section root, then tried to disambiguate inside the handler (long-vs-short drag, threshold tuning). It still leaked — swiping to spin the globe would also trip the swipe-end heuristic and advance a section.
